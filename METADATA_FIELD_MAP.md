# Metadata field map: Excel placeholders ↔ form JSON ↔ XML generator

This document ties together `**Metadata Placeholders and Script Generation.xlsx**`, the `**collectFormData()**` payload from `Index.html`, the server normalization in `**mapClientDataToServer**` (`ValidationSystem.gs`), and what `**UniversalXMLGenerator**` (`SchemaValidator.gs`) reads when building XML.

**Important:** The app emits **ISO 19115-2** (`gmd` / `gmi` / `gco`) only, with root `**gmi:MI_Metadata`**, aligned structurally with the NOAA/Navy UxS reference `**NOAANavyUxSAcquisition_MetadataTempate_19115-2-GMI_2026-2 (1).xml`** (NCEI `schema.xsd` in `xsi:schemaLocation`, `MD_Distributor`, `MI_CoverageDescription`, root `metadataMaintenance`, etc.). Field mappings below are about **data**, not every optional XML branch.

---

## 1. Payload shape

### Client (`collectFormData()` in `Index.html`)

```js
{ mission, platform, sensors, spatial, output }
```

### Server (`mapClientDataToServer` in `ValidationSystem.gs`)

- Spreads all keys from `clientData.mission` and `clientData.platform` through to the result.
- Overrides: `mission.id` ← `missionId` or `id`; `mission.title` ← `missionTitle` or `title`; cleans `abstract`, `contactEmail`, `contactAddress`.
- Overrides: `platform.id` ← `platformId` / `platformID` / `id` / `identifier`; `platform.name` ← `platformName` or `name`; `platform.description` ← `description` or `**platformComments**`.
- Passes through: `sensors`, `spatial`, `output` unchanged.

Use `**mission.id**` and `**mission.title**` in server-side code; the UI still posts `**missionId**` and `**missionTitle**`.

## 5. Known gaps / follow-ups

1. `**accessConstraints` free text**: The UI uses a textarea but XML maps it to `**MD_RestrictionCode`**; long prose may not match code-list validation.
2. `**mission.qualityStatement`**: Not used (data quality uses **spatial** accuracy fields).
3. **Strict structural checks** do not require temporal extent (only geographic `extent`), so records without acquisition dates still pass strict spine for that item.

---

## 6. Suggested JSON additions (minimal, Excel-aligned)

Add only what you need for NCEI validation:

- `mission.metadataDate`, `mission.purpose`, `mission.publicationDate`
- `mission.citeAs` / plain `**otherConstraints`**, `**mission.dataLicensePreset`**, `mission.licenseUrl` (custom preset only)
- `mission.aggregationInfo` (nested citations) or flat prefixed fields
- `spatial.verticalExtent`, `spatial.gridRepresentation.axes[]`
- `output.metadataLandingUrl`, `output.downloadUrl`, `output.distributionFormat`

Extend `**collectFormData()**` and, if aliases are required, `**mapClientDataToServer**`.
