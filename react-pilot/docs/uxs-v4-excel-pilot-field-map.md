# USX V4 Excel ↔ implementation map

**Canonical model:** `**react-pilot`** uses `**pilotState**` (mission profile) for editing, XML preview, import, and validation. That is the reference shape this project extends.

This table maps **USX V4**-style Excel and the **classic HTML wizard** (`missionId`, nested `output.*`, `gcmdKeywords`, …) onto that canonical model so Sheets round-trips and training docs stay understandable. Use it with [uxs-ncei-template-mission-pilot-matrix.md](./uxs-ncei-template-mission-pilot-matrix.md) for ISO path–level detail.

**Naming bridge (aliases → `pilotState`)**


| Classic wizard / Excel / legacy alias | `pilotState` (mission profile)                                                                                                                                        |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `missionId`                           | `mission.fileId` (merge also accepts `missionId` from loaded JSON)                                                                                                    |
| `missionTitle`                        | `mission.title`                                                                                                                                                       |
| `organization`, `contactEmail`        | `mission.org`, `mission.email`                                                                                                                                        |
| `nceiAccessionId`                     | `mission.accession`                                                                                                                                                   |
| `gcmdKeywords` / `gcmdKeywordsJson`   | `keywords.{sciencekeywords,datacenters,…}` as `{ label, uuid }[]`; for legacy `collectFormData`, `gcmdKeywords` is rebuilt from `pilotState` in `pilotStateToLegacyFormData` |
| `spatial.boundingBox` (corner object) | Prefer `mission.west/east/south/north`; a corner bbox is derived for `collectFormData` when axes are absent (`pilotToLegacyFormData.js`)                              |
| `platformComments`                    | `platform.platformDesc`                                                                                                                                               |
| `output.*`                            | Mostly `distribution.*`; `mergeLoadedPilotState` maps `loaded.output` onto `distribution` (see **Classic `loaded.output` merge** below)                               |


### Classic `loaded.output` merge (`mergeLoadedPilotState`)

All keys listed in `LEGACY_OUTPUT_DISTRIBUTION_KEYS` in `react-pilot/src/lib/pilotValidation.js` (the constant name is historical) copy **1:1** from `loaded.output` onto `distribution` when the value is not `undefined` / `null`. Additionally: `outputFormat` → `distribution.format`; `validationLevel` (`basic` / `strict` / `catalog`) → `mode` (`lenient` / `strict` / `catalog`) **only if** `loaded.mode` is absent; `metadataRecordDate` and `doi` on `output` fill `mission.metadataRecordDate` and `mission.doi` when those mission fields are still blank.

**Not merged** (intentional): `saveAsTemplate` (output-step UI only), `metadataSchema` (generator hint, no `pilotState` field).

### Aggregation + strict validation (scope)

**Aggregations:** Pilot uses flat `mission.parentProject*`, `relatedDataset*`, `associatedPublication*` → simplified `aggregationInfo` in preview/import paths; not full arbitrary-depth `CI_Citation` trees from the NCEI workbook.

**Strict vs legacy generator:** React `validatePilotState` / `missionValidationRules` and classic HTML wizard `SchemaValidator.gs` are **manually** kept aligned when rules change (no shared package yet). See [uxs-ncei-template-mission-pilot-matrix.md](./uxs-ncei-template-mission-pilot-matrix.md).
